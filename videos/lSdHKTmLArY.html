Basic flow: (1) payer makes HTTP request to service, (2) service returns 402 response with payment requirements, (3) payer signs authorization (no gas cost), (4) API/facilitator settles on-chain, (5) service returns product. Two payment schemes: (a) transfer with authorization (exact amount), (b) permit/spending cap (up to max amount, debit variable amount). Implementation: backend settle_payment() function before work, frontend use_fetch_with_payments() hook. Thirdweb's web facilitator uses project's own server wallet (not hosted), enabling real-time transaction visibility and fee capture. Multi-chain support: 170+ chains (Arbitrum, Base, Monad, Hyperliquid, Solana, etc.). Any token accepted (USDC default, custom tokens possible).
