Key sections: (1) Verify before you claim—label claims as "confirmed with real evidence" (file path, line number, actual command run) or "merely inferred"; trace real call chains, don't guess from code names, get test baselines before claiming no breakage; (2) Scope & Safety—stage only actual changes (never silently revert teammate work), name the rollback procedure before any destructive action (delete, migration, deploy), wait for user confirmation; (3) Communication—end every turn with honest status (commit hash, test counts, what was assumed vs. verified, what user must check locally). LEVER 2 — OpenRouter Fusion (Model Panel): Instead of betting on one model, convene a panel of up to 8 models answering in parallel. A judge model reads all answers and writes structured JSON comparing: where they agree, where they contradict, what each uniquely caught. Final answer built from best of all. Default preset: Claude Opus, latest GPT, Gemini Pro (three labs = three failure modes). Presets: "general high" (strongest), "general budget" (fast/cheap). Cost/benefit: trade extra cost for accuracy, which matters on critical tasks. Why it works: models fail differently. One hallucinates, another misses edge cases, a third nails it. Judge surfaces contradictions/blind spots explicitly.
