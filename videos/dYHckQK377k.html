Production architecture: S3 stays in place (no migration), ImageKit sits between client and S3, handles transcoding and HLS generation (50+ URL-based transformations), pushes chunks to global CDN. Implementation: ImageKit Next.js SDK wraps standard image/video components, adds URL generation and transformation logic. Features demonstrated: image cropping with AI focus=face (respect aspect ratio and safe zones), video thumbnails via IK-thumbnail parameter, HLS streaming via IK-master=m3u8 + transformation params (resolutions: 240p, 480p, 720p, 1080p), adaptive quality based on bandwidth. Code example: `<Image src={avatarPath} width={40} height={40} transformation={{focus: 'face'}} />` replaces manual cropping. Workflow: clone GitHub repo, sign up for ImageKit free plan (generous forever-free tier), create URL endpoint, update sample video paths, import ImageKit provider in layout, replace raw `<video>` and `<img>` with ImageKit components, add HLS params to video player (using hls.js library). Testing: on throttled 4G connection (Slow 3G), page loads lower-resolution variant automatically vs. full MP4 stall.
