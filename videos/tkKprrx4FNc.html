SYNOPSIS: Insomnia is Kong's open-source (Apache 2.0), cross-platform API client supporting REST, GraphQL, gRPC, WebSockets, Server-Sent Events. Core value proposition: API request collections stored as plain files in Git repos (not cloud), enabling version control, code review, and collaborative workflows. Local storage by default (no account required). Typical workflow: (1) Write request, extract moving parts into environments (base shared environment + dev/staging/prod sub-environments). Reference env vars inline with {{varname}} syntax. (2) Secrets (API keys) encrypted on device, masked in UI, never synced to cloud. (3) Switch between environments with one click (same requests, different values). (4) One-click migration from Postman (import collections, OpenAPI specs, HAR files, raw curl commands). (5) Git sync enabled: entire collection becomes plain files in repo. New endpoints appear as diffs, auth header changes reviewed in PRs. (6) Design-first workflow: OpenAPI spec becomes source of truth. Live preview available during editing. Collection of requests auto-generated from spec. (7) Inso CLI: integrates into CI pipelines. `inso lint spec` validates contract on each push. `inso run test` executes requests with real exit codes (enables gating deployments). (8) Self-hosted mock servers: Insomnia can spin up mock backends in Docker container on your own machine (avoids cloud dependency). Self-hosted mocks in enterprise tier only. Postman pricing: $9, $19, $49 per user per month. Five-person team on middle plan = $1,140/year. Insomnia + local Git storage = $0. 2023 trust incident: Kong released update requiring Kong account login to open app, locking users out of local collections. Community revolt. Kong walked it back in Insomnia 8.3; local mode is now default again. Bruno exists as a response—MIT licensed, promises zero cloud forever. Other design-first competitors: competing OpenAPI interpretation approaches, browser-based tools. Verdict: Free if you use local storage + Git. Paid enterprise tier for advanced features (self-hosted mocks, etc.). Headless operation via Inso CLI.
