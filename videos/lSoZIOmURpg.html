The ladder: (1) Does this need to exist? Skip speculative features. (2) Does standard library do it? Use it. (3) Native platform feature (CSS, DB constraint, date input)? Use it. (4) Already installed dependency? Use it; never add new. (5) Can it be one line? Use one-liner. (6) Minimum working code. Test results: Sort task—without Ponytail: 13-line hand-rolled quicksort (stack overflow risk), with Ponytail: 2 lines calling built-in sort (84% reduction, identical output). Email validation—without: 22-line regex class wrapper, with: 3 lines (86% reduction but permits invalid emails like bad.com; Ponytail docs note true email validation requires confirmation email). Four intensity modes: light (whisper suggestion), full (enforce ladder, default), ultra (YAGNI extremist—challenges necessity), off. Limitation 1: Prompt-dependent. Benchmark on Llama 3.2 local model showed 17-50% variance, no stable reduction. Works on Claude/GPT because those models already know the lazy move; weak models gain nothing. Limitation 2: Prose tax. Early versions wrote one-liner + long explanation; benchmarks showed near-parity tokens vs. baseline (total cost not per-line). Fixed by capping explanation length > code deleted explanation. Claimed "up to 77% cheaper" is best-case against 190-line-dashboard baseline. Hard guardrails Ponytail preserves: input validation at trust boundaries, error handling preventing data loss, security, accessibility, hardware calibration knobs.
