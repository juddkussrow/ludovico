Agents differ: agents are sessions that run indefinitely; workflows are transactions with start/end. Replay model wraps every step, caches execution history, replays on resume. Problem: replay log grows unbounded; agents doubling agent-run duration every 4-7 months already at hours, heading toward days. Solution: split durability. Context durability (append-only log of all LLM I/O) is trivial—store in database, object storage, or distributed filesystem. Execution durability (files, memory, subprocesses) cannot use logs; use snapshot-restore instead. Trigger.dev built on Firecracker: snapshot entire VM state, compress to 14 MB (seekable compression, lazy decompression on restore), sub-second snapshot time, ~100-200 ms restore. Historical precedent: IBM mainframes 1966 had checkpoint/restore; CRIU 2011 enabled user-space checkpoint of single process but can't capture non-process state (Chrome, FFmpeg). Firecracker allows full machine snapshot compatible with container runtimes. Benchmark: 15,000 VM starts/minute (TTI measure), achieves ~30 FPS if rendering video. FC Run (open source soon) is Docker-like CLI for Firecracker snapshots. Durable agents = context durability + execution durability; recover from LLM failures (wait, snapshot, restore later) or machine crashes (replay from context log).
