Pipeline: (1) CacheAligner — stabilizes context front for provider cache hits; (2) ContentRouter — routes chunks by type; (3) Content-specific compression: JSON compressor preserves anomalies/collapses repetitive arrays; AST pass on code keeps signatures/imports, folds bodies; log statistical analyzer keeps ERROR/WARN, drops INFO lines; prose analyzer ranks meaningful tokens via small LM. Test case: real 423-line log (35k chars, 2 errors, 1 warning, 420 passing-info lines) → 15 lines (96% compression), all errors+warnings preserved, run time <0.4s, no network. Breadcrumb: hash pointer left in place; headroom retrieve() expands original via second API call (potential footgun on small payloads where retrieval costs more than no compression). Protection: last-4-messages rule protects recent context (model's active reasoning); compression only on older tool outputs. On small fresh calls, compression saves 0% (by design). Benchmarks: 92% token cut (code search), 92% (SRE debugging), 73% (GitHub triage), 47% (codebase exploration); accuracy 0.870→0.870 (no drop on math/truthfulness). Works alongside Caveman (output-side compression). Install: pip install headroom-ai[all].
