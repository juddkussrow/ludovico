Pipeline flow: (1) User turn—transport input captures 20ms audio chunks, runs VAD (outputs user_started_speaking/user_stopped_speaking control frames), buffers until complete utterance detected. Speech-to-text (STT) processor outputs interim transcription frames (for UI) and full transcription frame (for LLM). Context aggregator builds universal context (role: user, content: transcribed text). (2) Assistant turn—LLM streams tokens, outputs LLMFullResponseStartFrame + LLMTextFrames + LLMFullResponseEndFrame. Text-to-speech (TTS) processor buffers at sentence/word boundaries (not immediate tokens), generates TTS audio raw frames + TTS text frames. Transport sends audio to speaker; context aggregator logs assistant response. (3) Loop repeats. Frame types: standard frames flow downward (top to bottom); control frames flow upward (enable state awareness downstream). Interruption handling: transport output sends bot_started_speaking control frame upward to transport input (maintains bot_speaking flag). VAD detects user speech during bot output → sends interruption_task control frame upward → pipeline task converts to high-priority frame, broadcasts downward → all processors cancel tasks, bot stops speaking. Processor logic: maintains standard queue + priority queue (checks if frame is high-priority, routes accordingly). VAD tuning: start_seconds (silence threshold before user_started_speaking), stop_seconds (silence duration before user_stopped_speaking), confidence threshold. Smart turn detector: outputs probability user finished thought; works with VAD to reduce false interrupts (VAD 700-800ms without smart detector causes interrupts during pauses; VAD 200ms + smart detector avoids this). TTS speak frame: allows bot to say fixed messages (welcome/farewell) at lifecycle hooks (on_user_joined, on_user_ready). STT mute filter: processor that conditionally mutes STT input; STTMuteStrategy.MUTE_UNTIL_FIRST_BOT_COMPLETE (mutes until bot's first utterance complete). Observers: external listeners on pipeline task, watch frame flow for telemetry/logging without modifying pipeline; must use async create_task to avoid blocking pipeline execution. Errata from transcript corrections: TTS buffers at sentence boundaries (not word boundaries as misspoken); STT mute strategy name is MUTE_UNTIL_FIRST_BOT_COMPLETE; event hooks on Transport (not PipelineTask); on_client_connected example.
