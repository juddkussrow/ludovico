Capabilities: (1) Text to video: enter prompt, generate clip. (2) Video to video: upload video + new prompt, map movements of reference video onto new scenario (e.g., barbarian as barbarian in winter, or woman as futuristic cyberpunk character). (3) Image-to-pose: upload image + pose skeleton video, animate image with pose movements; works for real people and anime. (4) Reference talking face to audio: upload face image + any talking/singing reference video, animates face to match audio+performance (body movement, background animation, natural expressions, no uncanny valley). (5) Audio generation: upload video + prompt (e.g., "birds chirp"), generate matching sound design. Web playground (requires CHN number); alternative: Replicate pay-per-use (~$0.07 per generation via H100 rental). Local install: ComfyUI + ComfyUI-HunyuanVideoWrapper by keyi (works on 12 GB VRAM with quantization/fp8 precision tradeoffs). Download transformer model (13 GB fp8 version for low VRAM) + VAE (bf16 version) into ComfyUI models folder. Run workflow: custom nodes manager → install Hunyuan Video Wrapper + VHS Video Helper → drag/drop workflow JSON → configure model/VAE dropdowns → run. Settings: width/height (default 512x320), frames (85 or custom, adjust for VRAM), inference steps (50 default, quality scales with steps, but diminishing returns >200), guidance scale, flow shift (2–4 for smaller videos, tunes motion), force offload (reduce VRAM). First run downloads additional files (~2–3 min); subsequent runs faster. Prompt testing examples: "astronaut riding unicorn in desert" (some foot-ground inconsistency but strong); "Pomeranian puppies learning to become chefs" (partial success, dogs not actively cooking vs. Minimax/Kling better); "Will Smith eating spaghetti" (generates unknown white male, fails celebrity recognition vs. Machi/Minimax success); "Northern lights timelapse" (realistic time-lapse, star/light timing correct except watermark artifact); "princess running from dragon, Disney Pixar style" (runs toward dragon instead of away, no video model achieved perfect version); "anime girl in Kyoto kimono" (anime style correct, girl standing not walking); "sad woman, red teary eyes" (excellent emotional expression, quality matches/exceeds commercial models); "POV soldier in war-torn city, high action" (chaotic but recognizable, some NPC disappearances, Kling generation superior); "zombie swarm attacking metro" (consistent, high-quality chaos, Hunyan preferred over Kling for this prompt). Quantization trade-offs: fp8/bf16 = lower VRAM but slight quality loss vs. fp32. Resolution at 512x320 shows occasional watermark artifact (rare, disappears on reroll). Leaderboard context: Hunyuan did not exist before this video; now competes with Machi (open-source), Minimax (commercial SOTA), Kling (commercial SOTA). Hunyuan matches or exceeds all three on many prompts; Minimax strongest on text rendering and celebrity recognition; Kling strong on high-action choreography. VERDICT: Hunyuan Video is a watershed open-source release. Free, uncensored, competitive with top commercial models on most prompts, runs locally on modest VRAM (12–16 GB). Weak points: celebrity recognition, text rendering, multi-character choreography (less consistent than Minimax/Kling). Strengths: emotional expression, landscape/nature generation, pose-guided animation, audio-driven animation, cinematic realism. Best value for users wanting local control + unlimited inference. Quantization/VRAM tuning required; no one-click guarantee. Top choice for open-source AI video as of late 2024.
