It uses Root/Stack mounting, Upsert singletons, and Mutation Flow for clean promise-based UI patterns.

Core mental model: call React components like async functions (similar native browser window.confirm API—code pauses, waits for user response, returns value inline). Eliminates state variables (is_modal_open), callback props (on_close, on_submit), context providers, and portals. Three architectural concepts: (1) Root & Stack—callable component acts as mounting point placed at app/layout root as listener; multiple calls managed via internal stack; renders all active instances, cleans up on close with exit animations, isolates instances. (2) Upsert—singleton UI pattern (global notifications, loading overlay, progress indicator); call with call() spawns new instance; call with upsert() updates existing if active, preventing duplicates; smoothly swaps data on-the-fly. (3) Mutation Flow—use_mutation_flow hook ties callable UI to async backend actions; manages pending state, shows loading spinner during async function execution, confirms UI completion only after promise resolves. Supports hot module replacement for fast dev workflow. Example walkthrough: confirm dialog uses create_callable HOC wrapper, on_click handlers for button functions; delete button invokes component asynchronously; no context, Redux, or Zustand needed. Complex examples available (async actions, nested calls). Result: cleaner, linear architecture replacing spaghetti callback/context code.
