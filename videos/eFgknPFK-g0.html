Traditional LLM decoding is memory-bound, generating one token at a time, leading to linear latency growth with output length. DSpark addresses this by using a small, fast "draft" model to propose a block of tokens, which a larger "target" model then verifies in a single pass. This process is lossless, producing identical output to the target model alone. The core latency equation is L = (T_draft + T_verify) / τ, where τ is the number of accepted tokens per round. DSpark improves this by: 1) drafting faster, 2) drafting better (higher acceptance rate), and 3) verifying smarter. Previous speculative decoding methods like autoregressive Eagle 3 are accurate but slow with small blocks, while parallel D-Flash is fast but suffers from "suffix decay" where tail tokens are often rejected. DSpark overcomes suffix decay with a semi-autoregressive draft head, which is a lightweight serial head bolted onto a parallel backbone, allowing each token to peek at the previous one. This results in DSpark accepting 30% longer blocks than Eagle 3 and 16-18% longer than D-Flash on Qwen models. Additionally, DSpark introduces confidence-scheduled verification to optimize resource usage under server load. It uses a confidence head to score tokens, and a hardware-aware scheduler verifies the full block under light load, but only the confident prefix under heavy load, skipping doomed tail tokens. DeepSeek has open-sourced DeepSpec, including training code and draft checkpoints, and uses DSpark in production for DeepSeek-V4-Flash and V4-Pro, achieving 57-85% faster per-user speed with the same total throughput. The method is model-agnostic, supporting models like Qwen and Gemma. The presenter attempted to replicate DSpark on a Mac M2 Max using a 0.6B draft model and an 8B target model with llama.cpp. While the acceptance patterns (high on code/reasoning, low on chat) matched the paper, the overall speedup was negative because the draft model was only 3.5x faster than the target, whereas 10-30x is needed for a speedup. This highlights the importance of using properly trained draft heads.
