Base workflow: UNETLoader (FP8 model) → CLIPLoader (Qwen3VL, type set to "Boogu") → EmptyLatentImage (resolution: 1024×768 default; portrait 832×1216 or 768×1344; landscape 1344×768) → CLIPTextEncode prompt → KSampler (Euler, Simple scheduler, 40-50 steps, CFG 4.0). Edit workflow: Load Edit model instead; TextEncodeBooguEdit node accepts original image + text instructions ("change clothes, keep person") + supports multi-image reference but single-image most stable; KSampler uses dpmpp_2m, Simple scheduler, 40 steps, CFG 3.5; ModelSamplingAuraFlow shift=3.16 (do not modify). Tested: 12 Base tests, 9 Edit tests. Base strengths: photorealistic foundation (documentary photos, animal close-ups render naturally), Chinese text and commercial posters (single/bilingual brand visuals, menu+prices), multi-panel layouts and infographics (outfit guides, hairstyle consistency, pet profiles), commercial product posters. Base weaknesses: small text, large text amounts, dense information, mathematical symbols (error rate increases). Edit model: clothes swaps (foreground replacement, face/pose/background preserved), background replacement (consistency excellent, lighting blends naturally), object addition (placement, reflections, shadows, occlusion), object removal (texture/shadow restoration).
