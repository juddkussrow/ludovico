Speaker Ido Salomon (creator of AgentCraft and MCP) diagnosed core bottleneck: engineers trying to manage reckless agents like human managers oversee employees, but engineers lack those soft-skill toolkits. Solution: borrow from RTS game design (real-time strategy) where players routinely manage dozens of units. AgentCraft implements four core mechanics: 1) Visibility: file-system-as-map metaphor; each directory is map sector; each file is room in room; agents working on files shown as visual heat map on map; can see collision prediction and lineage (which agent changed what, when). 2) Rapid reaction: RTS muscle-memory hotkeys to cycle between agents needing human approval (plan review, question answering). 3) Autonomy: "quests" feature lets human specify high-level goals; agents decompose, plan, execute within container without constant babysitting. Campaign orchestrator handles sub-agent coordination. 4) Collaboration: human-to-human (designer + engineer work in shared workspace, see each other's agents), human-to-agent (chat interface for both direct prompts and soft awareness of active changes), agent-to-agent (coordinated work on files). Advanced: cron job scheduling to auto-run agents on social media feeds (e.g., Twitter idea scanning) without human intervention; review bundles show all PR changes visually with screenshots/video evidence. Shift work from planning phase to review phase (multiple proposals reviewed vs single careful plan). Tool is free, experimental, available via QR code link and Discord for feedback.
