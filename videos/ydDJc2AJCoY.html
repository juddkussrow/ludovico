DSpark works by using a small, fast draft model to guess blocks of tokens ahead of time, which the larger model (e.g., DeepSeek V4 Pro, a 1.6 trillion parameter model) then verifies in a single pass. This process significantly reduces the GPU's idle time, which typically occurs while waiting for memory in autoregressive decoding. The output is mathematically identical to normal decoding due to rejection sampling, ensuring no loss in quality. DSpark improves upon older methods like Eagle (autoregressive) and DFlash (parallel) by combining a fast parallel backbone with a sequential Markov head for better acceptance rates across the entire block. It also incorporates a confidence head and a hardware-aware scheduler for zero-overhead scheduling, optimizing performance based on GPU load. This approach is particularly effective for coding agents because code is highly predictable, allowing the draft model to make accurate guesses for brackets, imports, and boilerplate. DeepSeek has open-sourced the entire training stack, called DeepSpec, under an MIT license, which includes three drafting algorithms and supports models like Qwen and Gemma, allowing users to train custom drafters. While training a drafter requires significant resources, running pre-trained checkpoints with vLLM or SG Lang is straightforward. The video demonstrates how to integrate DSpark with Claude Code by serving DeepSeek V4 via vLLM on localhost and using a thin proxy to adapt between Anthropic's message format (used by Claude Code) and OpenAI's format (used by the local server). This setup allows Claude Code to run tasks up to 85% faster on local hardware, reducing GPU compute costs.
