Wraps live websites into native desktop apps via Tauri 2 + system webview. App size: ~5 MB (film emulation project example built to 4.3 MB DMG, 55 MB installed). Electron equivalent: 310 MB (Slack comparison). Memory usage: 61 MB (Pake film tool) vs. Arc browser (much higher). Single command: `pake --url=<URL> --name=<AppName>`. Build time: 5–10 minutes. Flags: --debug (enables dev tools), --hide-title-bar (frameless), --inject-css <file>, --inject-js <file>, --show-system-tray (dock icon), custom tray icon. All features work without writing Rust (1,800 lines of Pake's own Rust on top of Tauri 2). Limitations: requires running URL (app blank if server offline); cannot edit default menu items; no local app bundling (unless --use-local-file flag). Build issue: newer PNPM version than Pake supports caused build failure; required Claude Code source-code edit. Pake compiled binary would resolve dependency issues. Alternatives: Electron Bun (Bun runtime backend, can run backend logic), Zero Native (Zig shell, C library calls). Use case: YouTube Music app (works offline because YouTube Music web app does). Not for locally-built desktop apps unless using --use-local-file. Not suitable for custom local desktop apps or complex backend needs; reach for Electron Bun or Zero Native instead.
