Rust crash course: install via rustup, use cargo init to scaffold projects, main.rs entry point. Essential crates added via cargo add: iroh (p2p protocol), tokio (async runtime), anyhow (error handling), rand (randomness), futures-lite (async streams), serde (serialization), clap (CLI args). Async patterns: use #[tokio::main] attribute for async main, result<T> for error handling, implicit returns (omit semicolon on last statement), use statements for imports. Endpoint setup: generate secret key (rand for randomness), bind endpoint with DNS discovery (ndz), pass secret key. Gossip protocol: enables message broadcast across arbitrary peer swarms. Topics isolate networks; peers join by dialing bootstrap nodes; gossip maintains 5 active peers + 20 passive peers per node (scales to tens/hundreds of thousands without central coordinator). Implementation flow: (1) Create topic ID from 32 random bytes. (2) Subscribe to topic via gossip.subscribe(topic_id). (3) Split receiver into sender/receiver tuple. (4) Broadcast messages: sender.broadcast(msg). (5) Listen for events: while loop over receiver.try_next(), match on GossipEvent. Important: gossip ignores own broadcasts locally to prevent self-dial loops and duplicate processing. Signaling (exchanging topic/peer info): use tickets—serialize topic ID + peer addresses into JSON, base32-encode, print to stdout. Other node pastes ticket, parses it, retrieves topic and peers to join. Roles: open (create room, generate random topic), join (accept ticket, use provided topic/peers). Message types: use serde for serialization. Define enum with variants (AboutMe { sender, nickname }, Message { sender, text }). Serialize to bytes as JSON, deserialize on receive, match on type to route logic. CLI via clap: derive-based argument parser, subcommands for open/join, required args (ticket for join), optional args (name for identifying peers in output). Example: `cargo run -- open` creates chat, prints ticket. `cargo run -- join <ticket>` joins room. Type in one terminal, see output on other. The code is incremental: compile, test, add feature, repeat. Compiler errors are learning moments. Full working example available in Iroh GitHub examples/gossip-chat; advanced version includes message signing with private keys to prevent impersonation.
