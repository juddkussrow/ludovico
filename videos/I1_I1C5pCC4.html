Architecture: controlled DSP sandbox safely compiles and swaps algorithms in milliseconds without crashing the host (solved the hard limitation of rewriting DSP code while real-time audio thread is running). Workflow: user describes plugin (e.g., "warm arpeggiated poly-synth that locks with DAW tempo"), pastes prompt into LLM (Claude, ChatGPT, Gemini, user's choice), pastes generated C Major code back into Amorph, hits compile. If compile fails, copy error, feed back to LLM, iterate (typically 1–2 iterations). C Major is a DSP language designed for audio with straightforward processor concept: audio input/output, parameters with ranges and names (auto-generate UI knobs). Supports MIDI input, mono/stereo audio in/out; enables audio effects, MIDI instruments, and hybrids. Presets: top-level presets are entire algorithms (code + knob names + device); inside each algorithm, save parameter snapshots (recall knob state without code change). Limitations: simple UI (knobs, readouts only; no fancy graphics/custom visualizers), not for ML models or sample libraries, focused on DSP (filters, modulations, synthesis, time-based effects, MIDI synthesis). Creator personally built ~30 different plugins; complexity ranges from basic filters to vinyl scratchers and realtime scrubbers. Requires specificity in prompts (ask for "stable variable filter 12dB/octave with resonance" not "filter that sounds good"). Community preset sharing planned. Pricing: free, open beta.
