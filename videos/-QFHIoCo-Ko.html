Core thesis: software engineering fundamentals (used for human teams) work equally well with AI. LLM constraints: (1) Smart/dumb zone—new conversation = smart zone (least attention strain); ~100k tokens = transition to dumb zone. AI degrades steadily after. Solution: keep tasks small, fit in smart zone. (2) Memento syndrome—LLMs forget; context resets. Prefer clearing over compacting. Token visibility critical (status line shows exact usage). Workflow phases (human-in-loop until implementation): Phase 1: Grill Me skill. Input client brief (Slack message: "retention bad, add gamification"). AI asks relentlessly about every aspect—points economy, retroactivity, progression curves, streak mechanics, UI placement. Result: shared design concept, alignment. Typical: 20-80 questions. Phase 2: Write PRD. Input grilling transcript. Output destination document (problem statement, solution, 18 user stories, implementation decisions, testing decisions, out-of-scope list). Phase 3: PRD to Issues. Transform PRD into Kanban board with vertical slices (tracer bullets). Key principle: AI loves horizontal slicing (all DB first, then API, then frontend)—bad because no feedback until phase 3. Vertical slices cross all layers (schema + minimal service + minimal UI = end-to-end feedback). Blocker relationships define parallelization. Phase 4: Implementation (AFK). Run ralph.sh script: passes all issues + last 5 commits + prompt to Claude Code with accept-edits permission in Docker sandbox. Agent picks next task (prioritizes critical bugs, infra, tracers, polish). Executes TDD: write failing test, implement, pass. Runs feedback loops (tests, type checks). Commits. Result: reviewable PR per issue. Key: small tasks fit smart zone; feedback loops essential; TDD prevents cheating (AI can't pre-implement if test is written first). Phase 5: Code review + QA. Parallel agents possible via separate worktrees. Architecture: system prompt (tiny), exploratory phase, implementation, testing. Clearing context = fresh smart zone for code review (dumber reviewer in dumb zone is bad pattern). Architectural design crucial: code quality → agent quality (garbage codebase → garbage agent output). Skills owned, model-agnostic. Token awareness (aiming for ~100k working room) essential. Demo: course management platform (CMS) adding gamification feature. Grill Me ran 22 questions auto. PRD generated 18 user stories. Issues split into 5 trackers. First issue (schema + minimal service + dashboard) executed with TDD, generated clean PR. Cost: human day shift (planning, grilling, PRD, issues), AI night shift (implementation AFK). Code review burden increases with volume. Not resolved: team feedback loops mid-execution, breaking upstream PRD changes, streaming vibe coding into production.
