Loopcraft is the skill of designing agentic loops—repeating structures where agents check reality, remember results, and retry—rather than one-shot prompts. The framework includes 5 loop levels (token, tool, goal, meta, team) and requires 4 conditions (repeating task, automated verification, budget absorption, senior tools) plus a gate preventing token waste.

Difference: prompt asks once; loop asks, checks result, remembers outcome, retries. Karpathy's autoresearch repo exemplifies this—agent edits train.py, runs fixed 5-minute experiment, checks validation score (val_bpb), keeps change if improved, discards if worse, repeats. Five loop levels: (1) token loop—model samples tokens until stop condition; (2) tool loop—agent calls tools until no more calls; (3) goal loop—agent runs, result judged, retries if needed; (4) meta loop—spawns multiple agents, reviews them, respawns better attempts; (5) team loop—allocates work across many agents. Real product proof: OpenAI defines Codex automation as scheduled/recurring Codex task. Anthropic describes agents as LLMs using tools based on environmental feedback in a loop. Anthropic's internal research shows multi-agent system (primary agent breaks query into subtasks, sends subagents in parallel) outperformed single Claude Opus 4 by 90.2% on research eval. Cost warning: 4x token usage vs normal chat, 15x for multi-agent; many coding tasks not parallelizable like research. Before building loop, run 4-condition test: (1) task repeats (amortize setup cost), (2) verification automated (something fails work without human), (3) token budget absorbs waste (loops reprocess, retry, explore), (4) agent has senior-engineer tools (logs, ability to run code, see breakage). Miss one condition = single good prompt wins. Minimum viable loop: 4 parts—(1) one automation (scheduled, stops on clear condition), (2) one skill (agent doesn't re-derive project from zero), (3) one state file (tomorrow resumes vs restart), (4) one gate (test/type-check/build failing bad work). Gate metric: cost per accepted change. If 50% of loop's changes rejected, loop became review task, defeating purpose. Ralph Wiggum loop failure mode: agent signals done early, loop exits on half-finished job, no hard stop, tokens burn. Internet divided: one camp says loops are real coding interface, other says token tax (only pays off unlimited plan). Both right depending on gate presence. Key takeaway: don't stop thinking/let agents run wild; move up a level—define objective, proof, boundary; build loop; stay engineer.
